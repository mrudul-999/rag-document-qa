## Real-Time LLM Streaming Pipeline (Architecture Evolution)

Goal: Stream LLM output in real time instead of waiting for a full response to reduce perceived latency and improve UX.

---

### Phase 1 Architecture: The "Queue & Callback" Method
(Initial Implementation)

**Flow:** LLM → Callback Handler → Cross-Thread Queue → FastAPI SSE → Frontend Generator → UI

**How it worked:**
* **Callbacks:** Used an `AsyncCallbackHandler` to intercept tokens as the LLM generated them.
* **Queues:** Pushed tokens into an `asyncio.Queue` acting as a buffer.
* **SSE:** `FastAPI StreamingResponse` continuously popped tokens from the queue and pushed them over the network via Server-Sent Events (SSE).

**The Problem:** Brittle. Bridging LangChain synchronous background threads with asynchronous FastAPI queues is prone to thread locking and missing <END_OF_STREAM> signals when using specific third-party APIs (like HuggingFace endpoints).

---

### Phase 2 Architecture: Native LCEL Streaming
(Current Production Implementation - The Big Refactor)

**Flow:** LLM Generator (`.astream()`) → FastAPI SSE → Frontend Generator → UI

**How it works now:**
* **LCEL (LangChain Expression Language):** We dropped the Queues and Callbacks entirely.
* **Direct Pipelines:** We explicitly format the prompt and invoke `async for chunk in llm.astream(formatted_prompt)`.
* **Zero Overhead:** The tokens yield directly from the underlying LLM async iterator straight into the FastAPI SSE stream.

---

## The Network & Client Layers (Constant across both phases)

### Server-Sent Events (Backend)
* `FastAPI StreamingResponse` keeps the HTTP connection open.
* Returns `content-type: text/event-stream`.
* Flushes data format: `data: {"type": "token", "content": "hello"}\n\n`

### Frontend Generator
* Connects to the `/stream` endpoint via `requests` using `stream=True`.
* Consumes the stream packet-by-packet using `.iter_lines()`.
* **Key mechanic:** Uses Python's `yield` keyword to return partial text chunks to Gradio.
* **Result:** Creates a smooth, fluid "typewriter" effect on the screen.

* Demonstrates awareness of modern Async paradigms (LCEL native streaming).
* Highlights network protocol knowledge (SSE vs WebSockets vs Polling).
